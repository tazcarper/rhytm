# Feature requests

Filled-in feature requests written by the client via the **`request-a-feature`** skill
(`.claude/skills/request-a-feature/SKILL.md`). Each file is one request, ready to hand to the
developer.

The question set behind them lives in `docs/feature-request-guide.md` (published for the client at
`/guide-feature-request.html`).

- One file per request, named `<short-slug>.md`.
- These are intake docs, not code — a developer turns them into an actual change.
