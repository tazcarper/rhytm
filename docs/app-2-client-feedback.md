# App 2 — Client Feedback Log

Capture every comment from the App 2 client walkthrough. Triage happens in this same file — the **Status** column carries the decision.

## How to log

Append rows to the table below. Don't worry about format polish — get the comment captured first, we'll clean up during triage.

**Type column:**
- `UX` — interaction, layout, friction
- `Copy` — wording, tone, naming
- `Bug` — something didn't work as expected
- `Feature` — "I'd love it if it also did X"
- `Brand` — visual / design / feel

**Status column:**
- `fix-now` — patch in App 2 before flipping to ✅
- `fix-in-App-3` — admin portal / staff-facing surface (App 3)
- `fix-in-App-6/7/8` — payments / e-sign / real email (downstream Apps)
- `defer` — keep on the radar, no commitment yet — promote to a deferred-improvements row when a trigger condition surfaces
- `wontfix` — explicit no, with reasoning
- `untriaged` — captured but not yet decided (default state when freshly logged)

If you're the client logging an item, leave **Status = untriaged**. We triage on our end and write back here.
