Contributor Setup

Make changes to the site

Set up your Mac to edit the Rhythm Outdoors website with Claude — by just talking to it. You don’t need to be a programmer, and you can’t break anything live.

Branch & preview Local & safe Reviewed before publish

iHow this works (60 seconds)

You describe a change. Claude does it on a safe copy, shows it to you instantly, and packages it up for your developer to publish.

  1. You ask. “Make the booking page hero bigger.”
  2. Claude works on a branch. A separate, safe copy — never the live site.
  3. You see it right away. On your own computer at http://localhost:3000.
  4. Claude opens a pull request. And shares a preview link you can pass around.
  5. Your developer reviews & publishes. That’s the safety gate — on purpose.
🛡️
You never touch the live database, never deploy, and can’t push to the live site. If you ask for something that would, Claude simply tells you it’s been handed to your developer — and keeps going with the rest.

1What you can & can’t do

✓ You can freely

  • Change layout, spacing, colors, fonts, styling
  • Edit text and content
  • Move existing sections and components around
  • Add new pages built from what’s there

→ Handled by your developer

  • Adding new software libraries or changing the app's foundation (build setup, login, how data is stored) — that's already built
  • Anything that changes the database (Claude writes it into your pull request automatically)
  • Publishing / deploying to the live site
  • Anything touching live payments or customer data

You don’t have to memorize this — the workspace enforces it for you.

2One-time setup

Do this once. If you get stuck on any step, paste the message to Claude and ask — it can help.

⚡
Fastest way — one command. First make a free GitHub account and accept your invite (step 2.1 below), and make sure you have Claude Code. Then open the Terminal app (press ⌘ + Space, type “Terminal”, press Enter).

Move into the folder where you want the project to live — if you’re not sure, your Documents folder is fine (the project lands in a rhythm-outdoors subfolder right where you are): 
cd ~/Documents
Now paste this single line — it installs everything, signs you in, downloads the project, and opens Claude for you: 
curl -fsSL https://rhytm-one.vercel.app/setup.sh -o ~/rhythm-setup.sh && bash ~/rhythm-setup.sh
⚠️ Run this inside the Terminal app. Don’t click the link or double-click a downloaded .sh file — that just opens it in a text editor. The command above downloads and runs it for you.

It pauses with clear instructions at the few steps only you can do (signing in, approving Docker the first time). Prefer to go step by step instead? Just follow 2.1–2.5 below.

2.1 — Get a GitHub account & accept the invite

  1. Sign up (free) at github.com.
  2. Send your GitHub username to your developer.
  3. They invite you to the project — check your email and accept the invitation.

2.2 — Install the tools

Open the Terminal app (press Cmd+Space, type “Terminal”, hit Enter), then run these one at a time:

Homebrew — an installer for the other tools. Paste it and follow the prompts:

/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

Node, Git, and the GitHub CLI:

brew install node@24 git gh

Docker Desktop — runs the practice database on your Mac. Download from docker.com/products/docker-desktop, install it like any app, then open it once so it’s running (look for the whale icon 🐳 in your menu bar). Leave it running whenever you work on the site.

🐳
What’s Docker for? It quietly runs a private copy of the website’s database on your own Mac, filled with sample data, so every page looks real while you design. Because it’s all local, you physically can’t touch real customer data. You never use Docker directly — Claude does.

Claude Code — you already have this and are logged in. (If you ever need to reinstall: npm install -g @anthropic-ai/claude-code.)

2.3 — Sign in to GitHub from the terminal

gh auth login

Choose GitHub.com → HTTPS → Login with a web browser and follow the prompts. This is what lets Claude save your work and open pull requests for you.

2.4 — Get the project onto your computer

gh repo clone tazcarper/rhytm
cd rhytm

2.5 — Let Claude finish setup

Start Claude inside that folder (type claude) and say:

“Set me up to work on this project for the first time.”

Claude installs everything, starts your local database, fills in your local settings, creates sample logins, and opens the site at http://localhost:3000. That’s it — you’re ready.

🔐
About settings & keys: your local settings are generated on your machine — no production secrets are ever sent to you. If your developer ever needs to give you a test key, they’ll send it as a one-time secret link, never plain email or chat. Never paste anything anyone calls “live” or “production.”

3Your day-to-day

Just talk to Claude. For example:

“On the homepage, make the hero image full-width and move the ‘Book Now’ button above the description.”

Claude starts a branch, makes the edit, shows it to you at localhost:3000, and iterates with you. Useful things to say:

  • “Show me that page” — makes sure the local site is running.
  • “I’m done with this one” — opens the pull request & shares the preview link.
  • “Start something new” — begins a fresh branch.

4Seeing your changes & finishing up

  • On your computer: http://localhost:3000 — instant, your main view.
  • Preview link: after Claude pushes, a preview of your branch builds automatically and Claude shares the URL — great for showing your developer.
💡
If a change involves the database, the preview link might not show it until your developer applies it — but your local site always shows it. Trust the local view.

When you’re happy, say “I’m done” and Claude opens a pull request. Your developer reviews and publishes it. You don’t merge or publish anything yourself — that’s the safety gate.

5If something goes wrong

You see…Do this
“Cannot connect to the Docker daemon” / database won’t startOpen Docker Desktop, wait for the 🐳 icon to settle, then ask Claude to try again.
Claude says something is “blocked”That’s the guardrail — that action belongs to your developer. Claude already captured it in your pull request. Just continue.
The site looks broken after getting new changesAsk Claude to “reset the local database and restart.”
Anything elsePaste the message to Claude and ask what it means.
